## RackNerd 延迟到底是什么

延迟，一句话解释：数据包从你设备出发到服务器再返回的时间，单位毫秒。越低越好。

交互类操作——SSH、远程桌面、网页浏览、实时应用——200ms 以内感觉是"即时响应"。过了 250ms 开始有粘手感。过了 300ms 就难受了。

RackNerd 在北美、欧洲、亚洲分布着 20 个数据中心，每个机房到你所在地的网络路径不一样。物理距离重要，但路由路径更重要——地理位置更近的服务器如果网络绕路，延迟反而更高。

## 下单前怎么测 RackNerd 延迟：五步流程

这步大部分人跳过，然后后悔。RackNerd 每个机房都公开了测试 IP 和 Looking Glass 工具。该用就用。

### 第一步：从你的网络 Ping 测试 IP

打开终端，对目标机房跑 Ping。关键机房测试 IP 如下：

- 洛杉矶 DC-02：`ping 204.13.154.3`
- 洛杉矶 DC-03：`ping 107.174.51.158`
- 圣何塞：`ping 192.210.207.88`
- 西雅图：`ping 192.3.253.2`
- 达拉斯：`ping 198.23.249.100`
- 纽约：`ping 192.3.81.8`
- 芝加哥：`ping 198.23.228.15`
- 阿什本：`ping 192.3.254.158`
- 亚特兰大：`ping 107.173.164.160`
- 阿姆斯特丹：`ping 23.94.101.88`
- 伦敦：`ping 185.239.172.14`
- 多伦多：`ping 167.160.186.8`

看两个数：平均延迟和丢包率。丢包超过 5% 就是红灯，就算 Ping 值好看也不行。

### 第二步：Traceroute 看实际路径

bash
# macOS / Linux
traceroute 204.13.154.3

# Windows
tracert 204.13.154.3


这会显示你到服务器之间每一跳的路径。如果你人在中国、目标洛杉矶，但路径绕到欧洲转了一圈，那就是路由问题。这种事确实会发生。

### 第三步：用 Looking Glass 测回程路径

每个机房都有 Looking Glass 页面，能从服务器端发起 Ping 和 Traceroute——也就是看回程路径。回程和去程经常不一样，很多"为什么延迟比预期高"的谜团都出在回程上。

Looking Glass 的地址格式统一：`lg-[机房代码].racknerd.com`。比如洛杉矶 DC-02 是 lg-lax02，圣何塞是 lg-sj，西雅图是 lg-sea，达拉斯是 lg-dal，纽约是 lg-ny。直接在浏览器输入对应地址就能用。

### 第四步：下载测试文件看实际带宽

每个 Looking Glass 页面都提供 100MB 或 1GB 的测试文件下载。拉一个下来看吞吐量。Ping 告诉你延迟，但文件下载速度告诉你带宽质量。有些机房 Ping 漂亮但晚高峰吞吐量塌方，实际体验一样差。

### 第五步：不同时段重复测试

凌晨两点的延迟不等于晚上九点的延迟。在你实际会用 VPS 的时段测。对中国用户来说，晚高峰（北京时间 20:00-23:00）是最考验网络的时候。如果这个时段机房扛得住，其他时段也没问题。

DC-02 是大部分 RackNerd 用户最终落地的地方，也是中文社区评测里出场率最高的机房。跑在 Multacom 动态路由网络上，路由会根据实时状况切换。

移动用户是三家里面最弱的。CMI 经香港中转带来波动，晚高峰波动超过 20%。

## 常见问题

**Q：为什么我买到的 VPS 实际延迟比测试 IP 高？**

测试 IP 跑在共享网络节点上，你的 VPS 有独立 IP，路由可能略有不同。另外测试 IP 通常在负载较低的网段。开通后从你的 VPS IP 跑一次 Traceroute 看真实路径。差距通常在 10ms 以内，但晚高峰可能更大。

**Q：买了之后能换机房吗？**

能。RackNerd 支持通过工单申请机房迁移。不是自助操作——你提交工单，他们的技术团队帮你搬。一般不收费，但取决于目标机房是否有库存。整个过程几小时到一天不等。

**Q：电信用户选哪个机房最好？**

圣何塞对电信用户延迟最低，双向直连。洛杉矶 DC-02 紧随其后，但晚高峰电信路径偶有丢包。如果你是电信，两个都测一下，哪个丢包低选哪个。

**Q：RackNerd 支持 CN2 GIA 等高端中国线路吗？**

不支持。RackNerd 用标准 BGP 路由，直连中国电信（163 骨干网）、中国联通（AS4837）、中国移动（CMI）。这些是常规直连线路，不是 CN2 GIA 那种高端优化线路。考虑到最低 $10.60/年的价格，这是合理的。要高端路由，你得看贵 5 到 10 倍的商家。

**Q：新加坡机房适合中国用户吗？**

看你的位置和运营商。新加坡到中国南方（广州、深圳）大概 60-80ms，但到北方可能 150ms+。中国到新加坡的路由不一定优化。对大部分中国用户来说，美西机房（圣何塞、洛杉矶）虽然绝对延迟更高，但表现反而更稳。

**Q：洛杉矶 DC-02 和 DC-03 有什么区别？**

是洛杉矶地区的两个不同物理设施。DC-02 跑在 Multacom 网络上，用动态路由，是更成熟的机房。DC-03 是较新的设施，用 ColoCrossing 基础设施，近期上了原生 IPv6 支持。从中国访问性能差不多，但 DC-02 的社区测试数据更多。两个都有货的话，DC-02 是更稳的选择。
